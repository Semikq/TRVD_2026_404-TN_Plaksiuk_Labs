## Опис моделей

### 🔐 Модель User
**Призначення**: Зберігає інформацію про користувачів та дані автентифікації
**Таблиця**: `users`

| Поле | Тип | Обмеження | Опис |
|------|------|------------|------|
| id | String | Первинний ключ | Унікальний ідентифікатор (cuid) |
| email | String | Унікальний | Email адреса користувача |
| password | String | Обов'язкове | Хешований пароль |
| firstName | String | Обов'язкове | Ім'я користувача |
| lastName | String | Обов'язкове | Прізвище користувача |
| phone | String | Необов'язкове | Номер телефону користувача |
| role | UserRole | За замовчуванням: CLIENT | Роль користувача (CLIENT/ADMIN/MANAGER) |
| createdAt | DateTime | Автоматично | Час створення акаунту |
| updatedAt | DateTime | Автоматично | Час останнього оновлення |

**Зв'язки**:
- Один-до-багатьох з `Order` (користувач розміщує замовлення)
- Один-до-багатьох з `Address` (користувач має адреси)
- Один-до-багатьох з `Review` (користувач пише відгуки)

---

### Модель Address
**Призначення**: Зберігає адреси доставки для користувачів
**Таблиця**: `addresses`

| Поле | Тип | Обмеження | Опис |
|------|------|------------|------|
| id | String | Первинний ключ | Унікальний ідентифікатор (cuid) |
| street | String | Обов'язкове | Вулиця адреси |
| city | String | Обов'язкове | Назва міста |
| zipCode | String | Обов'язкове | Поштовий індекс |
| isDefault | Boolean | За замовчуванням: false | Адреса за замовчуванням |
| userId | String | Зовнішній ключ | Посилання на User.id |

**Зв'язки**:
- Багато-до-одного з `User` (належить користувачу)
- Один-до-багатьох з `Order` (використовується для доставки)

---

### Модель Category
**Призначення**: Система категоризації продуктів
**Таблиця**: `categories`

| Поле | Тип | Обмеження | Опис |
|------|------|------------|------|
| id | String | Первинний ключ | Унікальний ідентифікатор (cuid) |
| name | String | Унікальний | Назва категорії |
| description | String | Необов'язкове | Опис категорії |
| imageUrl | String | Необов'язкове | URL зображення категорії |

**Зв'язки**:
- Один-до-багатьох з `Product` (містить продукти)

---

### Модель Product
**Призначення**: Каталог заморожених продуктів
**Таблиця**: `products`

| Поле | Тип | Обмеження | Опис |
|------|------|------------|------|
| id | String | Первинний ключ | Унікальний ідентифікатор (cuid) |
| name | String | Обов'язкове | Назва продукту |
| description | String | Обов'язкове | Опис продукту |
| price | Decimal | Обов'язкове | Ціна продукту (10,2) |
| imageUrl | String | Необов'язкове | URL зображення продукту |
| isAvailable | Boolean | За замовчуванням: true | Доступність продукту |
| weight | Integer | Обов'язкове | Вага в грамах |
| categoryId | String | Зовнішній ключ | Посилання на Category.id |

**Зв'язки**:
- Багато-до-одного з `Category` (належить до категорії)
- Один-до-багатьох з `OrderItem` (замовлені товари)
- Один-до-багатьох з `Review` (відгуки на продукт)

---

### Модель Order
**Призначення**: Замовлення клієнтів та їх деталі
**Таблиця**: `orders`

| Поле | Тип | Обмеження | Опис |
|------|------|------------|------|
| id | String | Первинний ключ | Унікальний ідентифікатор (cuid) |
| orderNumber | String | Унікальний | Номер замовлення (ORD-XXXXXXXX) |
| status | OrderStatus | За замовчуванням: PENDING | Статус замовлення |
| totalAmount | Decimal | Обов'язкове | Загальна сума замовлення (10,2) |
| deliveryType | DeliveryType | Обов'язкове | Тип доставки (PICKUP/DELIVERY) |
| deliveryFee | Decimal | Необов'язкове | Вартість доставки |
| promoCode | String | Необов'язкове | Застосований промокод |
| discount | Decimal | Необов'язкове | Сума знижки |
| notes | String | Необов'язкове | Примітки до замовлення |
| createdAt | DateTime | Автоматично | Час створення замовлення |
| updatedAt | DateTime | Автоматично | Час останнього оновлення |
| userId | String | Зовнішній ключ | Посилання на User.id |
| addressId | String | Необов'язковий зовнішній ключ | Посилання на Address.id |

**Зв'язки**:
- Багато-до-одного з `User` (розміщене користувачем)
- Багато-до-одного з `Address` (адреса доставки)
- Один-до-багатьох з `OrderItem` (містить товари)
- Один-до-одного з `Payment` (інформація про оплату)

| quantity | Integer | Обов'язкове | Кількість товару |
| price | Decimal | Обов'язкове | Ціна за одиницю (10,2) |

| productId | String | Зовнішній ключ | Посилання на Product.id |

**Зв'язки**:
- Багато-до-одного з `Order` (належить до замовлення)
- Багато-до-одного з `Product` (посилання на продукт)

**Зв'язки**:
- Один-до-одного з `Order` (належить замовленню)

### Модель Review
**Призначення**: Відгуки клієнтів та рейтинги продуктів
**Таблиця**: `reviews`

| Поле | Тип | Обмеження | Опис |
|------|------|------------|------|
| id | String | Первинний ключ | Унікальний ідентифікатор (cuid) |
| rating | Integer | Обов'язкове | Рейтинг (1-5 зірок) |
| comment | String | Необов'язкове | Текст відгуку |
| createdAt | DateTime | Автоматично | Час створення відгуку |

| productId | String | Зовнішній ключ | Посилання на Product.id |

**Зв'язки**:
- Багато-до-одного з `User` (написаний користувачем)
- Багато-до-одного з `Product` (відгук на продукт)

## Enum (Переліки)

### UserRole
- `CLIENT` - Звичайні клієнти
- `ADMIN` - Системні адміністратори
- `MANAGER` - Бізнес-менеджери

### OrderStatus
- `PENDING` - Замовлення розміщено, очікує підтвердження
- `CONFIRMED` - Замовлення підтверджено персоналом
- `PREPARING` - Замовлення готується
- `READY_FOR_PICKUP` - Замовлення готове до отримання/доставки
- `OUT_FOR_DELIVERY` - Замовлення у кур'єра
- `DELIVERED` - Замовлення виконано
- `CANCELLED` - Замовлення скасовано

### DeliveryType
- `PICKUP` - Самовивіз клієнтом
- `DELIVERY` - Доставка додому

### PaymentStatus
- `PENDING` - Оплата очікує обробки
- `COMPLETED` - Оплата успішна
- `FAILED` - Оплата не вдалася
- `REFUNDED` - Оповернення коштів

### DiscountType
- `PERCENTAGE` - Процентна знижка
- `FIXED_AMOUNT` - Фіксована сума знижки
